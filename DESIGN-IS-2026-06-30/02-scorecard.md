# 02 — Scorecard (продуктовая поверхность; 0–3 за принцип)

1. Good design is innovative — Score: 1/3
   Evidence: осознанный клон Claude Code (`DESIGN_BRIEF.md:2`, раскладка `App.kt:105`).
   Justification: имитация знакомого паттерна с мелкими доменными вариациями (чек-лист, токен-чипы) — не 0 (не копия «один-в-один»), но и не продвижение формы.

2. Good design is useful — Score: 3/3
   Evidence: пустое состояние + чипы-примеры (`App.kt:341`), Enter-отправка (`:376`), композер — главный элемент.
   Justification: основная задача (спросить→получить) делается в минимум шагов, без декой-действий на продуктовом экране.

3. Good design is aesthetic — Score: 2/3
   Evidence: дисциплинированная палитра/типографика (`Theme.kt:9`), но дрейф радиусов 6–20 и несистемный spacing + эмодзи-иконки (`01-evidence.md`).
   Justification: единая система видна, но ≤2 заметных рассинхрона (микро-шкала радиусов/отступов; эмодзи как иконки) — не 3, но и не 1.

4. Good design is understandable — Score: 2/3
   Evidence: продуктовые контролы ясны; «Инварианты» — жаргон (`App.kt:198`); композерные иконки без подписи (`:437,457`).
   Justification: один-два контрола требуют подсказки/переименования; критичные действия всё же опознаваемы → 2, не 1.

5. Good design is unobtrusive — Score: 3/3
   Evidence: тихий фон, лёгкие бордеры, контент — фигура, UI — фон (`App.kt:291–337`).
   Justification: хром уходит на задний план, ничто не конкурирует с содержимым.

6. Good design is honest — Score: 3/3
   Evidence: необратимость удаления (`App.kt:261`), маскирование ключей + «локально» (`Dialogs.kt:67`), честный баннер DNS (`App.kt:619`), нет тёмных паттернов.
   Justification: каждое заявление/ярлык соответствует поведению 1:1.

7. Good design is long-lasting — Score: 2/3
   Evidence: нейтральный минимализм Material3, без фейд-градиентов/скевоморфизма; но эмодзи-иконки (`App.kt:556,613,747`).
   Justification: один датируемый маркер (эмодзи-как-UI) — иначе считывалось бы свежо и через 3 года.

8. Good design is thorough — Score: 2/3
   Evidence: empty/loading/error/disabled присутствуют (`App.kt:341,885,330,417`); focus — дефолт Material; success — неявный.
   Justification: 1 состояние «грубое/по умолчанию» (focus) + нет явного success → 2.

9. Good design is environmentally friendly — Score: 2/3
   Evidence: веб-метрики N/A; idle-анимаций нет, спама нет; НО тёмной темы нет (`Theme.kt`), авто-скролл-анимация всегда вкл (`App.kt:311`).
   Justification: ресурсно-чисто, но мотив не гейтится reduced-motion и нет тёмной темы (нагрузка на внимание/глаза) → 2.

10. Good design is as little design as possible — Score: 3/3
    Evidence: продуктовый экран минимален — каждый элемент работает (`App.kt:291–413`).
    Justification: на продуктовой поверхности нет лишних/дублирующих аффордансов (dev-витрины вынесены в 00-scope).
