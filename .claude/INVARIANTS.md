# Инварианты состояния (День 14)

Правила, которые ассистент **не имеет права нарушать**. Реализовано по принципу **двойной защиты** из
роадмапа: инжект в промпт + пост-проверка стражем с перегенерацией в обоснованный отказ.

## Модель

`domain/Invariant.kt`:
- `Invariant(id, text, builtIn, active)` — правило на естественном языке.
- `BUILT_IN_INVARIANTS` — встроенные жёсткие правила консультанта (только законные пути; без юр. гарантий;
  оставаться в роли). Всегда активны, не удаляются.
- `renderInvariantsBlock(invariants)` — блок `[ИНВАРИАНТЫ — нельзя нарушать]` для системного промпта
  (только активные) + инструкция отказываться при конфликте и объяснять причину.

## Хранение — отдельно от диалога

Пользовательские инварианты: `accounts/<id>/invariants.json` (`data/InvariantStore.kt`). Встроенные — в
коде. Активный список = встроенные + пользовательские; держится в `ChatState.invariants`.

## Двойная защита

1. **Инжект в промпт.** Блок инвариантов добавляется в системный промпт **чата** (`ContextAssembler`)
   и **стадийных агентов задачи** (`TaskOrchestrator.invariants` → `call()`). Модель учитывает их в
   рассуждениях и сама отказывается.
2. **Страж** (`domain/InvariantGuard.kt`, дешёвый deepseek). После ответа агента проверяет его на
   нарушение активных инвариантов: `VIOLATION: …` или `OK`. При нарушении ответ **перегенерируется** в
   обоснованный отказ. Работает на пользовательских стадиях оркестратора (`TaskOrchestrator.call(guarded=true)`
   для intake / выполнения шага / ответа-ассистента) и в `VisaAgent.ask` (свободный чат). Стадии
   `proposeOptions`/`buildPlan`/`validate` не гонят стража (структурные/самопроверка — экономим вызовы).

## Поведение при конфликте

Запрос противоречит инварианту → ассистент **не предлагает** нарушающее решение, прямо отказывается,
называет нарушаемый инвариант и кратко объясняет; при возможности — законная альтернатива. Если модель
всё же «проскочила» — её ловит страж и заменяет ответ на отказ.

## UI

Окно **«Инварианты»** (кнопка в сайдбаре, `ui/Dialogs.kt::InvariantsDialog`): встроенные (только просмотр)
+ пользовательские (добавить / вкл-выкл / удалить). `ChatState`: `addInvariant / removeInvariant /
toggleInvariant` → сохранение в стор + обновление оркестратора.

## Карта файлов

- `domain/Invariant.kt` — модель, встроенные, `renderInvariantsBlock`.
- `domain/InvariantGuard.kt` — страж (пост-проверка).
- `domain/VisaAgent.kt` — инжект + страж + перегенерация в отказ (чат).
- `domain/Memory.kt` (`ContextAssembler`) — блок инвариантов в системном промпте.
- `domain/TaskOrchestrator.kt` — `var invariants` + инжект в стадийные запросы.
- `data/InvariantStore.kt`, `data/Dto.kt` (`InvariantDto`), `data/AccountStore.kt` (`invariants()`).
- `ui/ChatState.kt` — загрузка/управление, передача в агента/оркестратор.
- `ui/Dialogs.kt` — `InvariantsDialog`; `ui/App.kt` — кнопка в сайдбаре.
