# Антипаттерны

1. **Всё в один промпт** (профиль + история + факты + … = десятки k токенов) → модель теряет фокус,
   ответ деградирует.
   → Разделяй на слои, инжектируй только нужное для текущего шага (режим «3 слоя», `MEMORY_MODEL.md`).

2. **Решения за LLM в тексте** (переходы состояния / правила «просьбой» в промпте) → LLM услужливо
   согласится на что угодно.
   → Логику переходов и проверку инвариантов держать в **коде** (`require` / `check`), не в промпте (`STATE_MACHINE.md`).

3. **Постоянная суммаризация** → теряются детали.
   → Сворачивать историю редко; для новой темы — новый диалог (чистое окно лучше «болота» из всего подряд).

4. **Молчаливое проглатывание ошибок API** (показать «пустой ответ» вместо причины).
   → Поднимать понятную ошибку (статус + текст), как в `data/LlmClient.kt` (например, 429 rate-limit).
