# Промпт-инжиниринг

Суть: роль + цель + ограничения + формат + примеры сужают «бесконечный» контекст знаний и поднимают
качество. Уровни: simple → pro → pro-max (мета-промптинг).

## Системный промпт агента

`VISA_SYSTEM_PROMPT` (в `domain/VisaAgent.kt`) — единый источник (DRY). Задаёт:
- **роль** — визовый специалист;
- **ограничения** — не выдавать юридические гарантии; помечать, что нужно сверять с офиц. источником;
- **формат** — пакет документов оформляется отдельным блоком:

```
[checklist]
- Загранпаспорт; нужен
- Фото 35×45; нужен
[/checklist]
```
статусы только: `нужен` / `загружен` / `проверен`.

## Рендер формата

UI парсит блок `[checklist]…[/checklist]` (`ui/App.kt → parseSegments` → `ChecklistView`) в карточку
со статусами и цветами (`StatusColors`): нужен · загружен · проверен · не хватает.
