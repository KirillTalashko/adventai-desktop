# Персонализация и аккаунты (День 12)

Персонализация поверх памяти: **профиль предпочтений** (как отвечать) + **локальные аккаунты** (изоляция данных).

## Профиль vs память

| | Профиль (Day 12) | Память (Day 11) |
|---|---|---|
| Что | предпочтения: КАК отвечать | факты: ЧТО известно |
| Откуда | задаёт пользователь (онбординг) | агент извлекает сам из диалога |
| Где | `accounts/<id>/profile.json` | `memory/*.md`, `working/*.json` |

Оба инжектируются в каждый запрос; профиль идёт первым блоком `[ПРОФИЛЬ ПОЛЬЗОВАТЕЛЯ]`.

## UserProfile (`domain/UserProfile.kt`)

- **name**, **about** (контекст: «еду впервые, турист»);
- **length**: Кратко / Сбалансированно / Подробно;
- **tone**: Дружелюбный / Деловой;
- **formats** (мульти): Чек-листы / Пошагово / Без воды / С примерами;
- **constraints**: «только бесплатные источники», «я новичок», …;
- **language**.

`toPromptBlock()` рендерит инструкцию для system prompt; подмешивается в `ContextAssembler` всегда.

## Аккаунты (`data/AccountStore.kt`)

Локальные, **без пароля/сервера**. У каждого аккаунта — изолированные данные; ключи API общие.

```
~/.adventai/
  config.json                 ключи API + модель (общее)
  accounts.json               список аккаунтов + активный
  accounts/<id>/
    profile.json              профиль предпочтений
    conversations/ + index.json
    working/<convId>.json
    memory/profile.md + decisions.md
```

## Онбординг (`ui/Onboarding.kt`)

При первом запуске (нет аккаунтов) или при «Новый аккаунт» — экран `ProfileForm` (общая форма для онбординга
и редактирования, `ui/ProfileForm.kt`). По завершении создаётся аккаунт + сохраняется профиль.

## UI

- Сайдбар: **переключатель аккаунтов** (`AccountSwitcher`) — список, «Профиль…», «Новый аккаунт».
- **ProfileDialog** (`ui/Dialogs.kt`) — отдельное окно редактирования профиля.
- `ChatState`: активный аккаунт → per-account репозитории (`conversations`/`memory`/`profile`); переключение
  пересобирает их. Профиль передаётся в `VisaAgent.ask(... , profile, ...)`.

## Проверка («разные профили → разные ответы»)

Создай два аккаунта с разными профилями (напр. «Кратко, без воды, я профи» vs «Подробно, дружелюбно,
я новичок, с примерами»), задай один вопрос — ответы заметно различаются по стилю/детализации/формату.
