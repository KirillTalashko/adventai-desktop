## 🧠 Философия курса (зачем «ковыряемся в кишках»)

- **Готовые гайды на YouTube почти бесполезны**: устаревают за месяцы, плюс к тебе перетекают все *biases* автора. Теория без практики выветривается за неделю.
- Цель — **не копировать чужие инструменты, а уметь создавать свои** (или хотя бы понимать, почему чужие работают и какие из них рабочие).
- *«Странно учить сложносочинённые предложения до изучения алфавита»* — идём от голого API к MCP/RAG/пайплайнам.
- Аналогия **1890 года**: автомобиль — удел гиков, мастерских нет, сломалось — чинишь сам. Сейчас с ИИ так же.
- Различие: *не разбираться потому что **экономишь ресурсы*** (оправдано) vs *потому что **лень*** (не оправдано).
- **«Собери свой субъективный опыт взаимодействия с нейронками — другого способа научиться этому скиллу сегодня нет»**; стандартов в отрасли почти нет.

## 🔌 Работа с LLM API

- **Только прямые API вендоров** (DeepSeek/GigaChat/OpenAI/Anthropic). Никаких агрегаторов-упрощаторов: они режут/чистят контекст, скрывают параметры (макс. токены, температура) — и ты потом не учитываешь то, чего «не видел».
- **Нет стандартизации даже внутри одного вендора**: breaking changes ~раз в полгода. Температура где-то «выше = креативнее», а где-то наоборот.
- **`reasoning_effort` / efforts (Claude)** — отдельный рычаг качества reasoning.
- Остановка диалога-сбора данных, два способа: (а) заполняешь сущность, все поля `!= null` → стоп; (б) промптом «как собрал всё нужное — остановись». Можно сделать оба и сравнить.
- **Свой HTTP-клиент руками писать смысла нет** — Ktor/Retrofit/OkHttp ок. Главный вопрос себе: *«понимаю ли я, КАК это сделано?»*

## 🗂️ Память и контекст

- **Три слоя памяти**: долговременная (напр. «все проекты — только Kotlin»), рабочая (память проекта/задачи), сессионная (текущая работа). На каждый — своя папка; хранилище — **файлы** (если БД — NoSQL). По сути **уровневый CLAUDE.md**.
- Память заполняется **и юзером, и LLM-кой автоматически** (фоновый агент периодически корректирует mem-файлы) — «большие дядьки делают и так, и так».
- **Context leveling**: саммари диалога разбивается на уровни — стратегический / тактический / локальный — и хранится отдельно (по таймеру или событиям). В Claude это «из коробки»: глобальный/проектный/локальный.
- **Деление одного окна по топикам**: систем-промпт определяет тему → сопоставляет с файлом из списка (передаём только *названия + краткое описание + облако тегов*, не сами файлы) → подгружает контекст нужного топика. Отдельный инстанс может следить, что тема не сменилась. Уменьшает дорогую и вредную для контекста **саммаризацию** — «дроби ветки, чем больше тем лучше».
- ⚠️ Прямой совет под наш проект: *«важно, что это должен быть именно ЧАТ (история накапливается), а не серия one-shot запросов»*.

## 🤖 Агенты и оркестрация

- **Определение агента**: *«ограниченный скоуп модели под конкретную задачу»*. Аналогия — **агент = класс в ЯП**, у него есть инстансы; общие черты: входные/выходные контракты, system prompt, зарегистрированные tools. *«Вокруг этого построен Koog»*.
- **Агент ≠ профиль/настройки/state management**. Аналогия: агент — гоночная тачка; конфиги/память/профили — асфальт, правила гонок, пит-стопы.
- **Каждому агенту — только свой контекст**. Тащить всю историю в систем-промпт = захламление, **тупиковый немасштабируемый путь**.
- **Пайплайн** напр. `Research → Create → Validation`, на каждой стадии свой system prompt и набор агентов. Параллельные агенты → оркестратор собирает заключения → передаёт дальше. Бывает **оркестратор оркестраторов**.
- **Переходы между стадиями лучше детерминированные** (родился артефакт → следующая стадия): проще и надёжнее, чем доверять переходы нейронке. Отсюда же — **паузы**: на сомнении/валидации спрашивать юзера (флаг «спрашивать между этапами»).
- **Рой агентов**: на стадию 5 агентов с разных сторон + взаимопроверка (3 проверялы, которые спорят). Это **пчелиный улей/муравьиная колония, не рекурсия**.
- **Инварианты**: эвристика — *«когда нейронка выдаёт то, от чего „ну как так можно писать“ — кандидат в инвариант»*.
- **Fable 5 хорошо оркестрирует сам** → тест: если после включения Fable 5 резко стало лучше, значит обвязка была настроена плохо.

## 🔗 MCP

- **Демистификация**: *«MCP — это просто тот же скоуп модели со встроенным HTTP-клиентом для забора внешних данных, и всё»*.
- **Анализ/саммаризация всегда на стороне LLM. Tool только поставляет данные** (как датчик: системные вызовы → отдать данные).
- **Экономика токенов (главное)**: при работе с MCP нейронка **каждый раз тянет всю схему инструментов**. Разница в потреблении **~×100**. Один большой MCP вместо нескольких — расход растёт «в 2 раза за каждый запрос списка тулов», причём не все тулзы нужны.
- Задание «3 MCP» — это про **3 тула в одном клиенте**, не 3 сервера. Пример: «найди лучший ресторан в Питере и сохрани в заметки» → один прогон: поиск + отчёт + сохранение.
- Для гугла/ноушна **своё писать не нужно** — бери готовое; своё пишут под доменную специфику.
- **Публичные MCP для тестов**: Inspector `kite-mcp-inspector.fly.dev`; Text-Extractor / Time / Everything / Echo на `*.mcp.inevitable.fyi/mcp`.

## 💰 Модели: что под что (сетап ведущего)

- *«90% задач спокойно ведутся на Haiku»* — спеки, автотесты, релизы, общение с маркетами.
- **Написание кода — Sonnet.**
- **Opus — только для интерпретации исследований** (<10% работы).
- **Не накатывай Opus на всё**: правильно — подобрать **оптимальную модель под задачу** (golden-standard). Под техподдержку ≠ Opus.
- Секрет эффективности (анонс): **`caveman + ast-index + low effort + profiles + старые модели`** — но старые модели без state management/profiles дают «херню».
- **«Дешёвые закончились в 2025»**, подписка $100 — базовый минимум; на API уйдёт пара баксов, основное — подписки. $200 подписка ≈ $8000 в токенах; Claude работает в убыток.

## 🔒 Безопасность / opsec

- **Ресёрч-репост**: китайская схема перепродажи «Claude-токенов» за 5% цены. Платишь за Opus — получаешь Haiku или вообще не-Anthropic. Главный риск: **логируют все твои запросы целиком** (промпты, thinking, код, tool-calls) → на этом учатся чужие модели. По данным Anthropic утекло 150k диалогов в DeepSeek, 3.4M в Moonshot, 13M в MiniMax.
- **Вывод**: *«Никогда не отсылай чувствительные данные ни в какие прокси/дешёвые подписки. Даже OpenRouter — сомнительно. Только прямые API вендоров, лучше — свой сетап в своём контуре. If you don't own your AI, your data is owned by someone else»*.
- **Готовься к KYC в нейронках** (как в крипте); проверка паспорта «не за горами».
- **OpenRouter с 27 июня заблокировал РФ-аккаунты** (по гео-IP, обходится ненадёжно).
- **Не клади секреты на GitHub**: env-файлы под gitignore; для видео-демо продуктовых вещей — отдельный проект (чистка ранних коммитов не спасает от продуктовых утечек).
- Доступ к API из РФ: **DeepSeek/GigaChat без VPN**; `proxyapi.ru` (префиксы пути `/openai/`, `/anthropic/`, `/google/`); казахская/именная карта; VPN не выключать.

## 💼 Карьера

- Собесы **полностью AI-ориентированы**: в Яндекс (AI Marvel) — 1 час собрать агентскую систему (парсинг + внешние сервисы). На втором этапе отваливается ~50%.
- *«Завайбкодил» на хороших грейдах не прокатит* — конкурируешь с теми, кто реально знает.
- Из выпускников: грейдапы, смена работы с ×2 зп, «локальные звёзды», хорошо на perf-review.
- **`ast-index`, `Claude-in-mobile`, «консилиум»** — родились у участников прошлых челленджей «из необходимости».
- 🎯 **Наш проект похвалил лично**: визовый нейроассистент — «норм тема, консультанты берут неадекватно много, я часто визы оформляю, воспользовался бы». Ачивка «Эффективный менеджер»; проход назвал «самым экономным по токенам».

## 🔭 Что впереди

- **Курс = 7 недель / 35 дней.** Впереди: локальные модели (отдельная неделя — «облачные сильно отличаются», мощности не лютые), RAG / vector embeddings (с локальными), пайплайны. **«К 7 неделе всё выстраивается в одну картину»**, 7-я — самая интересная по сложности.
- **«Возможно 9 поток — последний».**
- После финиша — **Advanced**: execution loop, fine-tuning, **LLM security** (игра «взломать чужую модель» через prompt/indirect injection; защиты у большинства ботов слабые).
- **Claude по подписке как API**: поднять Claude на удалённом сервере по подписке и ловить ввод/вывод не через `claude -p` → использовать почти как API (есть готовый «маркетплейс решений»). Флаги пропуска подтверждений: Claude `--dangerously-skip-permissions`, Codex `--yolo`.
