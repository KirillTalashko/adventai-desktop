# SKILL: visa-cli (локальный помощник «Визового специалиста»)

Это навык работы с **локальным CLI** `visa-cli`. В отличие от MCP-инструментов (их схемы грузятся в каждый
запрос), эта инструкция подключается **по требованию** — только когда навык включён и реально нужен.

## Когда применять
Используй `visa-cli`, когда задача про **локальные данные пользователя**, которые не нужно искать в интернете:
- посмотреть, какие документы пользователь уже приложил.

## Как вызвать
Чтобы выполнить команду, выведи ОТДЕЛЬНОЙ строкой:

```
[CLI] visa-cli <команда> [опции]
```

Система выполнит её и вернёт результат строкой `[CLI_RESULT] …`. После этого дай человеку обычный ответ,
опираясь на полученные данные. Можно вызвать несколько команд по очереди (вывод одной — вход рассуждения для
следующей). Аккаунт подставляется автоматически — `--account` указывать не нужно.

## Доступные команды
- `visa-cli docs` — список приложенных документов аккаунта (имя + размер).
- `visa-cli docs check` — содержимое документов (извлечённый текст PDF) для СВЕРКИ данных.
- `visa-cli version` — версия CLI.

## Проверка достоверности (важно)
Когда нужно проверить документы, вызывай `visa-cli docs check` и по извлечённому тексту СВЕРЬ:
- на ОДНО ли лицо оформлены все документы (одинаковые ФИО/имя владельца);
- бьются ли даты (бронь/билеты/страховка) с датами поездки.
Если документы похоже на РАЗНЫХ людей или данные противоречат — ЯВНО предупреди (⚠️) и НЕ принимай пакет
как валидный. Если у файла текст не извлёкся (скан) — скажи, что проверить автоматически не вышло.

## Правила
- Вызывай ТОЛЬКО `visa-cli` (других команд нет; произвольный shell запрещён).
- Не выдумывай содержимое файлов — отвечай строго по выводу `[CLI_RESULT]`.
- Если по выводу видно, что документов нет, — так и скажи и подскажи, что приложить.
